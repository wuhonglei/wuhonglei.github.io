windows+ubuntu双系统

2016-04-03
windows+ubuntu双系统

windows+ubuntu双系统

系统


安装环境：老式台式机，新购240g固态硬盘SSD，打算将光驱的sata线插在SSD上。
问题：在开机机情况下，直接将光驱位sata线插在SSD上，电脑上不显示盘符（在磁盘管理中也无法查看)

盘符无法显示的原因：
　　1. SSD无盘
　　2. 老主板不支持无盘符的硬盘

解决方法:
　　1. 将SSD插在较新电脑的光驱位上然后 （不用关机）
　　2. 用分区工具将SSD分区或者直接在磁盘管理中将SSD格式化为mbr格式（gpt可以支持2T以上的硬盘容量，一般建议选择mbr分区合适）；

SSD分区完成后:
　　1. 将SSD插在旧主板的光驱位sata口上，然后重启系统（否则仍然无法读取），就可以识别盘符了。
　　2. 用diskgenius对SSD进行分区，因为要在SSD上安装windows系统，所以务必要分一个主分区。

安装windows系统至SSD主分区:
　　1.将u盘制作成U盘启动盘（可以用大白菜或深度等工具制作）
　　2.重启系统进入winpe，选择安装系统至SSD的主分区上；安装完成后重启，可能这时候仍然进入的是以前的windows系统：
原因: 主板上的sata接口有sata0,sata1,sata2，电脑在监测可以引导的启动盘时也会按照这样的顺序去引导（如果在sata0所连接的硬盘上，找到可引导的系统，那么电脑将会进入该系统，不会检查sata1所连接的设备是否可引导）
解决办法: 调换主板上机械硬盘和SSD的sata数据线位置，这样就能正确进去SSD的windows系统，为了避免通过互换sata线去正确引导SSD上的系统，我们可以用diskgenuis分区工具，将机械硬盘的系统分区“取消激活”，这样就不会再引导了机械硬盘上的系统了。

安装ubuntu至固态硬盘的另一分区：
　　1. 我们先进去安装好的windows，在磁盘管理中删除用来安装ubuntu的分区，这样做是为了等会安装ubuntu时，能分区成功。
　　2. 先将ubuntu镜像刻录到u盘中，可以使用ultra iso 、win32 diskimage或者Universal USB Installer
　　3. 将刻录好的Ｕ盘插在电脑上，然后重启电脑，选择Ｕ盘启动
　　4. 启动时如果屏幕出现 Failed to load COM32 file menu.c32 boot: _，需要输入live，屏幕上会提示我们是try ubuntu or install ubuntu ， 记住要选择try ubuntu, 进入ubuntu系统后我们需要对SSD的ubuntu分区进行格式化，格式化方法是打开terminal终端，然后输入gnome-disks命令，这时磁盘格式化工具就会跳出来，我们对SSD上的ubuntu分区进行格式化，格式化完成后安装ubuntu。
　　5. 分区方式：将SSD上的120G作为ubuntu的分区，具体分给ubuntu的大小是：

分区 格式 类型 大小 描述
/ ext4 primary 20G 15～20G
/boot ext4 logical 400M 200M就够了
swap swap area logical 8G 内存大小的两倍
/home ext4 logical 90G 剩余的空间

　　6.分区完成后，就可以选择安装系统了，安装正常步骤就可以安装成功
